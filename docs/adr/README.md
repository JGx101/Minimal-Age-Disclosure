# ADR Process

Use Architecture Decision Records to capture major design choices.

## When to create an ADR
Create an ADR when you are deciding:
- whether to adopt a standard or protocol
- how to model claims
- how to handle status / revocation
- whether to use pairwise identifiers or another anti-correlation mechanism
- whether to accept a tradeoff that affects privacy, security, or regulator fit

## ADR format
Use the template in `ADR_TEMPLATE.md`.

## Numbering
Use zero-padded numbering:
- 0001-title.md
- 0002-title.md
