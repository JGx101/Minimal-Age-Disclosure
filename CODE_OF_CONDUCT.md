# Code Of Conduct

## Purpose
This repository is a public design and standards project. Productive disagreement is expected. Personal attacks, dismissive behavior, and low-effort disruption are not.

## Expected behavior
Participants are expected to:
- critique ideas, requirements, threat models, and tradeoffs rather than people
- support strong claims with sources, concrete examples, or explicit reasoning
- keep discussion focused on privacy, security, governance, interoperability, and testability
- acknowledge uncertainty where the architecture is still unresolved
- respect the project's narrow scope: minimal age disclosure, not general identity collection

## Unacceptable behavior
The following are not acceptable:
- harassment, intimidation, or discriminatory conduct
- personal attacks, insults, or ridicule
- bad-faith argumentation, deliberate derailment, or repetitive spam
- requests to weaken the normal-flow privacy boundary without explicit technical or governance justification
- publishing private vulnerability details contrary to [SECURITY.md](SECURITY.md)

## Scope
This code of conduct applies to:
- issues
- pull requests
- discussions
- review comments
- other project communication channels tied to this repository

## Moderation
Maintainers may:
- ask participants to change tone or narrow scope
- hide, lock, or remove disruptive content
- close conversations that are no longer productive
- restrict participation for repeated or serious violations

## Reporting
If conduct in this repository becomes abusive or unsafe, contact the maintainer privately where possible and include:
- the location of the interaction
- a short description of what happened
- any relevant screenshots or links

For security or privacy vulnerabilities, use the process in [SECURITY.md](SECURITY.md) instead.
