# Support

## How to engage
This repository is best suited to:
- architecture and standards critique
- privacy and security review
- conformance and testability feedback
- policy and governance feedback
- issue reports about broken links, inconsistent docs, or unclear repository structure

## Where to ask what
- Use GitHub Discussions for open-ended questions, design discussion, and directional feedback.
- Use GitHub Issues for concrete defects, contradictions, and scoped proposals.
- Use [SECURITY.md](SECURITY.md) for private reporting of vulnerabilities or sensitive privacy failures.

## What makes feedback useful
Useful support requests usually include:
- the document, ADR, or section involved
- the exact ambiguity, contradiction, or risk
- why it matters for privacy, security, governance, or interoperability
- a suggested correction, alternative, or source if available

## What this repository does not provide
This repository does not currently provide:
- implementation support for production systems
- legal advice
- certification or compliance determinations
- a maintained software release line
